# AGENTS.md — Shadowsocks Client

> Windows-only desktop Shadowsocks client built with **Wails v2 + Go**.
> The webview window embeds a single-file SPA (no build toolchain). **All UI↔Go traffic uses Wails bindings (`window.go.main.App.*`) and Wails events (`runtime.EventsOn`) — there is NO HTTP API, NO npm step.**
> Includes an embedded **GeoLite2-Country mmdb** for automatic CN/GeoIP-based split routing (domestic direct, foreign via SS).

## Project Identity

- **Type**: Wails v2.12.0 + Go 1.25 desktop app (Go backend + WebView2 frontend)
- **Binary**: `build\bin\shadowsocks-client.exe` (~8MB after UPX; contains ~9MB embedded mmdb)
- **Title**: "Shadowsocks"
- **Window**: 640×790, non-resizable, dark `#0F0F1A` background; minimises to system tray on close

## Build & Run (Windows PowerShell)

```powershell
# Development — Wails hot reload (launches webview, watches .go + frontend/)
wails dev

# Build release — 一键编译 + UPX 压缩
.\build.ps1

# 仅编译不压缩（调试用）
wails build
# → build\bin\shadowsocks-client.exe (~20MB, no UPX)
```

`build.ps1` 会先 `wails build`，再调用 UPX `--best` 压缩到 ~8MB。UPX 路径：`$env:USERPROFILE\upx\upx-4.2.4-win64\upx.exe`。

Go module requires **Go 1.25+** (toolchain auto-downloaded by go 1.26 if `go.mod`'s `go 1.25.0` is higher than local). After editing imports, run `go mod tidy`.

Sanity check only (does NOT produce a working GUI binary):
```powershell
go build ./...
go vet ./...
```

## Architecture

```
main.go → wails.Run(options.App{
            Assets: //go:embed all:frontend   ← embedded single-file SPA
            Bind:   []any{app}               ← 7 methods exposed to JS
          })

app.go → App
          ├─ startup:   load cfg, proxy.New(), bridge logs → Wails events
          ├─ shutdown:  client.Stop() (restores system proxy) + cfg.Save()
          ├─ ctx       context.Context
          ├─ cfg       *config.AppConfig
          └─ client    *proxy.Client

main.go → single.TryLock("Shadowsocks") → Windows Named Mutex 防止多实例
```

### Communication channels

| Surface | Direction | Mechanism | Used for |
|---|---|---|---|
| `window.go.main.App.X` | JS → Go | Wails auto-generated bindings in `frontend/wailsjs/go/main/` | UI actions (status / save / delete / switch / toggle / quit) |
| `runtime.EventsEmit("log", msg)` | Go → JS | Wails event bus | Real-time log stream from the proxy engine |

**No HTTP server for UI.** `internal/config.HttpPort` (default 8899) is the **local HTTP proxy listener**, not an API port.

## Packages

### `internal/proxy` — proxy engine

- **`proxy.go`**
  - **Log bus** (package-level): `SubscribeLogs() chan string` / `UnsubscribeLogs` — buffered(20) pub/sub. `app.go` subscribes in a goroutine and forwards messages to `runtime.EventsEmit("log", ...)`. **No SSE anywhere.**
  - `Client` struct: `Start(server, port, password, method, httpPort, socksPort)` / `Stop()` / `Running()`.
  - `handle(client)` (HTTP CONNECT + plain absolute-URI GET/POST) and `handleSOCKS(conn)` both: parse target → check `isLanHost` → either **direct connect** or `net.DialTimeout(ssAddr)` → `cipher.StreamConn` → `relay`.
  - Crypto borrows only `core.PickCipher` + `cipher.StreamConn` from `go-shadowsocks2`; **HTTP CONNECT and SOCKS5 handshakes are hand-rolled** (no `net/http` parsing).
- **`bypass.go`** — LAN bypass layer (NOT a stub, NOT China-CIDR based):
  - `parseSSTarget`, `isLanHost`, `isLanIP` — decide which targets bypass SS
  - `directRelay` / `directRelayHTTP` — unencrypted relay to LAN target (TCP only)
  - `relay(conn1, conn2)` — shared bidirectional `io.Copy` + WaitGroup; used everywhere (proxy + bypass)
- **`system.go`** — Windows registry (`HKCU\…\Internet Settings`) system proxy on/off + bypass list. `SetSystemProxy(port)` backs up then enables; `RestoreSystemProxy()` writes the backup back.

### `internal/config` — `~/.shadowsocks-client/config.json` (0600, dir 0700)

```go
type ServerConfig struct { Name, Server, Password, Method string; Port int }
type AppConfig struct {
    HttpPort   int  // 默认 8899 — 本地 HTTP 代理监听端口（非 API）
    SocksPort  int  // 默认 1080
    Servers    []ServerConfig
    ActiveIdx  int  // -1 = 无
    AutoStart  bool
}
```
`Load()` returns `DefaultConfig()` (no error) when file is missing. `Save()` writes immediately.

## Bindings (`app.go` → `frontend/wailsjs/go/main/App.d.ts`)

Regenerated by `wails build`. **After editing `app.go` methods/types, always rebuild with `wails build`** (not `go build`).

| Method | Signature | Purpose |
|---|---|---|
| `GetStatus` | `() StatusResponse` | running + active idx + ports + servers |
| `SaveConfig` | `(httpPort, socksPort int) error` | update local proxy ports |
| `SaveServer` | `(in ServerInput) error` | add (idx<0) / edit server by idx |
| `DeleteServer` | `(idx int) error` | delete server; stops proxy if active removed |
| `SwitchServer` | `(idx int) error` | stop + switch active + start |
| `ToggleProxy` | `() (ToggleResponse, error)` | start/stop on active node |
| `Quit` | `()` | exit the app |

Shared types live in `app.go`: `StatusResponse`, `ServerInput`, `ToggleResponse`. Sentinel errors in `errors.go`: `errInvalidIdx`, `errNoServers`.

## Frontend (`frontend/index.html`)

- Single-file vanilla CSS/JS/HTML SPA (no framework, no npm). **No build step.**
- Backed calls go through a local `call(fn, ...args)` helper wrapping `window.go.main.App[fn]` (handles multi-return `(T, error)` and "binding-not-ready" retries).
- Logs are consumed via `runtime.EventsOn('log', cb)`; `runtime.js` is loaded via `<script src="/wails/runtime.js">` in `<head>`.
- Form read/write is centralized via a `formFields` map + `setForm`/`getForm` helpers (add a field → one line, edit/clear/save all follow).

## Conventions

- **Chinese comments** throughout Go and JS — match style.
- **JSON tags `snake_case`** on all config structs; exported field names follow Go conventions.
- **Logging is dual-channel**: notable events call both `log.Printf` (visible only when launched from a terminal / `wails dev`) **and** `addLog(msg)` (visible in the UI log panel). Prefer `addLog("[代理]/[系统]/[直连] ...")` so users see it.
- **Concurrency**: shared structs (`Client`) carry their own `sync.Mutex`. Lock briefly; copy fields out before I/O. Per-connection goroutines with `WaitGroup`.
- **Config saves immediately** on every mutation (`cfg.Save()` right there — no dirty layer).
- **Compact code style**: one-line `if err != nil { ... }` and `mu.Lock(); defer mu.Unlock()` are idiomatic; don't expand unless asked.

## Bypass / Split-routing rules

Decision order in `shouldBypass(host)` (any match → **直连** / direct):

1. **LAN / loopback / private**: `isLanIP` (10/172.16-31/192.168/127/169.254/::1), `localhost`, `*.local`, dot-less hostnames (`nas`, `printer`)
2. **CN domain suffix**: ends with `.cn`, or `.com.cn` / `.net.cn` / `.org.cn` / `.gov.cn` / `.edu.cn` / `.ac.cn` / `.sh.cn` / `.bj.cn` / `.gd.cn` (see `isCNDomainSuffix`)
3. **CN GeoIP** (via embedded GeoLite2-Country mmdb):
   - IP literal → `isChinaIP` directly
   - Domain → 2s-timeout DNS lookup; any returned IP `isChinaIP` → direct
4. **Otherwise** → proxied via SS

- DNS lookup failure → **保守走代理** (avoid unknown traffic bypassing the tunnel)
- mmdb load failure → **降级**: CN GeoIP disabled, routing falls back to LAN-only (all non-LAN → proxy)
- Log prefixes: `[直连]` direct, `[代理]` proxied, `[系统代理]` registry changes, plus `[GeoIP]` for db status on startup
- `ProxyOverride` registry value also covers LAN so system-apps don't tunnel local traffic: `127.0.0.1;localhost;<local>;10.*;172.16.*...172.31.*;192.168.*;*.local`

## Pitfalls

1. ⚠️ **`wails build` is mandatory, not `go build`**. `go build` skips binding regeneration (`App.js`) and asset embedding — UI will break silently (no `window.go.main.App` methods).
2. ⚠️ **Process name == exe file name** (Windows). `outputfilename: "shadowsocks-client"` → process is `shadowsocks-client.exe` / Task Manager shows `shadowsocks-client`. Renaming the file changes the process name; you cannot keep two different names without self-copy/restart tricks.
3. ⚠️ **System proxy via Windows registry** (`internal/proxy/system.go`), NOT PowerShell. `SetSystemProxy` writes `ProxyEnable=1` + `ProxyServer=127.0.0.1:<HttpPort>` + `ProxyOverride` LAN list.
4. ⚠️ **✕ / Alt+F4 hides to tray** (`HideWindowOnClose:true` in `main.go`), does NOT exit. The only clean exit is the tray menu → "退出" → `runtime.Quit` → `OnShutdown` (stops proxy, restores registry, saves config) → `systray.Quit`. Task Manager *End Task* / `taskkill /F` skips cleanup.
5. ⚠️ **`log.Printf` is invisible in the release GUI** (no console). Use `addLog()` for user-facing messages. To see stderr during dev, launch from a terminal (`wails dev` or `& path\to\shadowsocks-client.exe`) or wrap: `cmd /c "shadowsocks-client.exe 2>&1"`.
6. ⚠️ **CGO is mandatory** on Windows for Wails/WebView2 + systray (`energye/systray` uses `user32.dll`). `CGO_ENABLED=0` will break the build.
7. ⚠️ **`geoip2-golang` v2 API differs from v1**. Import path is `github.com/oschwald/geoip2-golang/v2`. Use `OpenBytes` (not `FromBytes`), pass `netip.Addr` (not `net.IP`), read field `ISOCode` (not `IsoCode`). Mixing v1-style code will not compile.
8. ⚠️ **Get the module name right**: it is `oschwald` (not `oschian`/`oschwald` variants). Wrong author name produces confusing 404s on goproxy.cn.
9. ⚠️ **mmdb binary lives at `internal/proxy/data/GeoLite2-Country.mmdb`** (~9MB). Do not move it — `geoip.go`'s `//go:embed data/...` is resolved relative to that file's directory.
